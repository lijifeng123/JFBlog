# 手写博客文章发布流程

## 方式一：手动发布（传统方式）

### 步骤 1：创建文章文件
在 `source/_posts/` 目录下创建 markdown 文件，例如：`我的新文章.md`

### 步骤 2：编写文章内容
文章格式如下：

```markdown
---
title: 文章标题
date: 2025-12-26 20:00:00
tags: 标签1, 标签2
---

# 文章标题

这里是文章正文内容...
```

### 步骤 3：提交到源仓库
```bash
cd /Users/lijifeng/Desktop/JFBlog
git add source/_posts/我的新文章.md
git commit -m "发布新文章: 我的新文章"
git push origin master
```

### 步骤 4：生成静态文件
```bash
npx hexo generate
```

### 步骤 5：部署到 GitHub Pages
```bash
npx hexo deploy
```

这会自动：
- 将生成的静态文件推送到 `lijifeng123.github.io` 仓库
- GitHub Pages 会自动更新网站

## 方式二：使用 MCP 自动发布（推荐）

### 步骤 1：准备文章内容
准备好 markdown 格式的文章内容（可以包含或不包含 front matter）

### 步骤 2：通过 AI 助手发布
直接对我说："发一篇博客，内容如下：..."

我会：
1. 自动保存文件到 `source/_posts/`
2. 自动提交到源仓库（JFBlog）
3. 自动生成静态文件
4. 自动部署到 GitHub Pages

### 步骤 3：确认发布
我会先显示预览，你确认后才会真正发布。

## 重要说明

### 两个仓库的区别

1. **源仓库** (`JFBlog`)
   - 地址：https://github.com/lijifeng123/JFBlog
   - 内容：源代码（markdown 文件、配置、主题等）
   - 用途：备份和管理源代码

2. **部署仓库** (`lijifeng123.github.io`)
   - 地址：https://github.com/lijifeng123/lijifeng123.github.io
   - 内容：生成的静态 HTML 文件
   - 用途：GitHub Pages 实际使用的文件

### 为什么在 GitHub 上看不到新文章？

- 如果查看的是 `lijifeng123.github.io` 仓库：这里只有 HTML 文件，看不到 markdown 源文件
- 如果查看的是 `JFBlog` 仓库：需要确保新文章已经推送到这个仓库

### 检查新文章是否已推送

```bash
# 检查源仓库状态
cd /Users/lijifeng/Desktop/JFBlog
git status

# 如果有没有提交的文件，执行：
git add source/_posts/你的文章.md
git commit -m "发布新文章"
git push origin master
```
